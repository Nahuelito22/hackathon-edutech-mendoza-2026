# Proof of Concept (Prueba de concepto)

## Idea general
Esta carpeta contiene dentro diversas carpetas con sus respcetivos nombres de la ia que creo la prueba de concepto.
Ej: ChatZ crea una prueba de concepto y se creara la carpeta ChatZ y dentro el html que responde a ello.

Con esto queremos manejar diversas ideas de ia para no estancarnos con una sola y ver cual nos conviene mas y por donde comenzar.

## Promt utilizado:
Actúa como un Diseñador UI/UX Senior y Desarrollador Frontend Experto.


Tu objetivo es crear una Prueba de Concepto (PoC) en un único archivo HTML (con CSS y JS embebidos) para la plataforma web oficial de la Hackathon EduTech Mendoza 2026 – 2da edición.



1. Contexto del Proyecto:

Organizadores: IES 9-023 de Maipú y el IES Tomás Alva Edison.   

Temática transversal: Desafíos actuales de la educación.  

Fechas clave: 3 de junio (Virtual), 5 y 6 de junio (Presencial).  

Sedes: Espacio Cultural Julio Le Parc y Escuela Edison.
  

2. Requerimientos Técnicos:

Utiliza HTML5 semántico.

No utilices frameworks externos (ni Tailwind, ni Bootstrap). Todo el CSS debe ser puro (Vanilla CSS) utilizando CSS Variables (:root) para la paleta de colores.

Crea un diseño completamente Responsive (Mobile First).

Incluye un Panel de Control flotante (con JavaScript básico) que permita al usuario cambiar en tiempo real entre tres paletas de colores predefinidas.



3. Paletas de Colores a implementar (Temas):

Tema 1 (Cyber Dark): Fondo principal negro/gris muy oscuro, texto principal blanco, color primario azul neón, color secundario verde brillante, color de acento naranja.

Tema 2 (Eco Light): Fondo principal blanco, fondo secundario gris claro (#f5f5f5), color primario verde bosque, color secundario azul marino.

Tema 3 (High Contrast): Que mezcle un fondo oscuro elegante con los tonos azul y verde académico de las versiones claras.



4. Estructura de la Landing Page:
Genera las siguientes secciones con contenido simulado (Lorem Ipsum o texto referencial):

Navbar (Sticky): Logo (texto "EduTech 2026"), enlaces a "Inicio", "Ediciones Anteriores", y botones destacados de "Iniciar Sesión" y "Registro".

Hero Section: Título principal de gran impacto, bajada de texto explicando que es la 2da edición, fechas del evento, y un Call to Action (CTA) principal: "Inscribirse ahora".

Sección 'Acerca del Hackathon': Tres columnas o tarjetas explicando la Temática (Educación), las Sedes (Le Parc y Edison) y la Modalidad (Equipos de 3 a 5 personas).

Sección 'Ediciones Anteriores': Un diseño tipo "Podio" destacando al ganador del primer lugar (puedes usar el nombre "Proyecto Guidia" como ejemplo de ganador del año pasado), seguido de una grilla estilo cards para otros proyectos destacados.



Footer: Links a "Bases y Condiciones", "Acerca de los Creadores" y Copyright.

Entregable: Proporciona únicamente el bloque de código completo HTML/CSS/JS listo para ser copiado, guardado como .html y ejecutado en el navegador.

Además agrego que tienes libertad absoluta para crear secciones que creas acordes, puedes usar otras herramientas siempre y cuando todo quede en un HTML para facilidad, y la paleta de colores pude variar respecto a esos diseños antes mencionados o crea un controlador para modificar eso según las paletas de colores que veas mas optimas para el proyecto según las posibilidades de logos que tenemos.

Te envie:
2 variantes de logos que nos dieron:
1) variante 1 fondo transparente
2) variante 2 fondo transparente
Tenemos 4 variantes de logo pero como todas manejan los mismos colores solo te envio esas dos para que mas o menos tengas el contexto.

Ademas te envio archivo
* Bases y condiciones: que marca un poco la estructura del proyecto y como se encamina.
* Form de inscripcion: Es un txt donde pase el formulario actual que tienen, que seria el que migraremos a la nueva pagina.
